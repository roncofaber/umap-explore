# UMAP Explorer — Design Spec
**Date:** 2026-07-13  
**Status:** Approved

---

## Overview

An interactive web app for a summer school audience (scientists, not necessarily coders) to explore how UMAP parameters affect dimensionality reduction results. Users select a dataset and adjust parameters via sliders/dropdowns; the visualization updates instantly with smooth animated transitions between embeddings.

---

## Architecture

Two-component system:

- **`precompute.py`** — offline script run once (or once per new dataset). Computes UMAP embeddings for all parameter combinations using `umap-learn` with `random_state=42`. Saves results as one JSON file per dataset under `data/embeddings/`.
- **`app.py`** — FastAPI server. On startup, does nothing heavy — embeddings are served on demand from disk. Also serves the static frontend.

No database. No runtime UMAP computation. All interaction is: fetch pre-computed JSON → re-render Plotly plot.

## Project Structure

```
umap-explore/
├── precompute.py           # offline precomputation script
├── app.py                  # FastAPI entry point
├── datasets/
│   ├── __init__.py         # dataset registry: name → loader function
│   ├── iris.py
│   ├── mnist.py
│   └── swiss_roll.py
├── data/
│   └── embeddings/         # generated by precompute.py, one JSON per dataset
│       ├── iris.json
│       ├── mnist.json
│       └── swiss_roll.json
├── static/
│   ├── index.html
│   ├── app.js
│   └── style.css
└── requirements.txt
```

Adding a new dataset: add a loader in `datasets/`, register it in `datasets/__init__.py`, run `python precompute.py --dataset <name>`. No changes to `app.py` or the frontend.

## Parameter Grid

Pre-computed for every combination of:

| Parameter | Values |
|-----------|--------|
| `n_neighbors` | 5, 10, 15, 20, 30, 50 |
| `min_dist` | 0.0, 0.05, 0.1, 0.25, 0.5, 1.0 |
| `n_components` | 2, 3 |
| `metric` | euclidean, cosine, manhattan, correlation |

**Total per dataset:** 6 × 6 × 2 × 4 = **288 embeddings**

All computed with `random_state=42`.

## Data Format

Each `data/embeddings/{dataset}.json` is keyed by a parameter tuple string:

```json
{
  "15_0.1_2_euclidean": {
    "x": [1.2, 0.4, ...],
    "y": [0.3, 1.1, ...],
    "z": null,
    "labels": [0, 1, 0, ...],
    "label_names": ["setosa", "versicolor", "virginica"]
  },
  "15_0.1_3_euclidean": {
    "x": [...], "y": [...], "z": [...],
    "labels": [...], "label_names": [...]
  }
}
```

`z` is `null` for 2D embeddings. Labels are integer indices; `label_names` maps them to display strings. For continuous labels (Swiss Roll), `labels` contains float values and `label_names` is `null`.

## API

| Endpoint | Description |
|----------|-------------|
| `GET /` | Serves `static/index.html` |
| `GET /static/{file}` | Serves CSS/JS assets |
| `GET /api/datasets` | Returns list of available datasets with metadata |
| `GET /api/embeddings/{dataset}?n_neighbors=15&min_dist=0.1&n_components=2&metric=euclidean` | Returns embedding for that parameter combination |

`/api/datasets` response:
```json
[
  {"name": "iris", "label": "Iris", "n_points": 150, "has_labels": true},
  ...
]
```

`/api/embeddings/{dataset}` response: the matching entry from the JSON file (x, y, z, labels, label_names).

## Frontend UI

Single-page, two-column layout:

**Left panel — controls:**
- Collapsible "What is UMAP?" section with a 3-sentence plain-language explanation
- Dataset dropdown
- `n_neighbors` slider (discrete steps matching the grid)
- `min_dist` slider (discrete steps matching the grid)
- `n_components` toggle button (2D / 3D)
- `metric` dropdown
- Brief tooltip/description beneath each control

**Right panel — visualization:**
- Plotly.js scatter plot (`scattergl` for 2D, `scatter3d` for 3D)
- Points colored by class label (categorical colorscale) or continuous value (sequential colorscale)
- Hover shows label name and point index
- Small loading indicator during fetch

**Transitions:**
- Parameter changes (n_neighbors, min_dist, metric, n_components) use `Plotly.animate()` with `transition: {duration: 400, easing: 'cubic-in-out'}` for smooth coordinate interpolation between embeddings
- Dataset changes (different point count) use a full `Plotly.react()` re-render (no animation)
